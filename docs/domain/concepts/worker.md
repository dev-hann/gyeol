# Worker (워커)

## 정의

워커는 레이어에 소속된 **AI 에이전트**입니다. 실제로 LLM을 호출하여 태스크를 수행하는 실행 주체입니다.

각 워커는 자신만의 시스템 프롬프트, 사용할 모델, 생성 온도 등의 설정을 독립적으로 가집니다. 같은 레이어에 여러 워커가 있으면, 레이어로 들어온 태스크를 워커들에게 병렬로 분산 처리합니다.

## 핵심 속성

### 이름 (Name)
워커의 고유 식별자입니다. 시스템 전체에서 중복될 수 없습니다.

### 소속 레이어 (Layer)
모든 워커는 반드시 하나의 레이어에 소속되어야 합니다. 독립적으로 존재할 수 없습니다.

### 시스템 프롬프트 (System Prompt)
워커의 역할과 지침을 정의하는 프롬프트입니다. 예: "당신은 코드 리뷰어입니다. 입력된 코드의 품질을 평가하고 개선점을 제시하세요."

### 모델 (Model)
워커가 사용할 LLM 모델입니다. 지정하지 않으면 시스템 기본 모델을 사용합니다. 레이어 내에서 워커마다 다른 모델을 사용할 수 있습니다. 예: 정밀한 분석은 GPT-4o, 빠른 분류는 Llama3

### 생성 온도 (Temperature)
LLM 응답의 무작위성을 조절합니다. 낮을수록 일관된 응답, 높을수록 다양한 응답을 생성합니다. 미지정 시 시스템 기본값(0.7)을 사용합니다.

### 최대 토큰 (Max Tokens)
LLM 응답의 최대 길이를 제한합니다. 미지정 시 시스템 기본값(4096)을 사용합니다.

## 프롬프트 계층 (Prompt Hierarchy)

워커가 실행될 때 프롬프트는 계층적으로 조합됩니다:

```
┌─────────────────────────────────────┐
│ 1. Context Prompt (스레드)          │  ← 전체 실행 컨텍스트
├─────────────────────────────────────┤
│ 2. Layer Prompt (레이어)            │  ← 레이어 공통 지침
├─────────────────────────────────────┤
│ 3. System Prompt (워커)             │  ← 워커 고유 역할
└─────────────────────────────────────┘
         ↓ 조합 (순서대로 병합)
┌─────────────────────────────────────┐
│ 최종 시스템 프롬프트                │
└─────────────────────────────────────┘
         + 사용자 메시지 (태스크 정보)
         ↓ LLM 호출
┌─────────────────────────────────────┐
│ 워커 실행 결과 (WorkerResult)       │
└─────────────────────────────────────┘
```

이 계층 구조 덕분에 스레드-레이어-워커 각 수준에서 독립적으로 프롬프트를 관리할 수 있습니다.

## 실행 결과

워커가 실행되면 두 가지 결과를 낼 수 있습니다:

- **성공**: LLM 응답을 담은 새로운 태스크를 파생시킴. 이 태스크는 큐에 다시 들어가 후속 처리됨
- **실패**: 에러 메시지를 기록. 필요시 재시도

## 활성화 상태

워커는 활성/비활성 상태를 가집니다. 비활성 워커는 레이어가 태스크를 분산할 때 제외됩니다.

## 다른 개념과의 관계

- **Layer**: 워커는 반드시 하나의 레이어에 소속됨. 레이어로 들어온 태스크를 분배받음
- **Task**: 워커가 태스크를 수행하고, 결과로 새 태스크를 파생시킬 수 있음
- **Provider**: 워커가 LLM을 호출할 때 제공자 설정을 사용하여 실제 API에 접근
- **Thread**: 스레드 실행 시 프롬프트 계층에 컨텍스트 프롬프트가 추가됨
