# Task (태스크)

## 정의

태스크는 Gyeol 시스템이 처리하는 **최소 단위의 작업**입니다.

사용자나 시스템이 하나의 처리를 요청하면 태스크가 생성되어 큐에 들어가고, 스케줄러가 이를 꺼내어 적절한 레이어로 라우팅합니다. 워커가 태스크를 수행하고, 결과로 새로운 태스크를 파생시킬 수 있습니다. 이런 파생 구조가 체인을 형성하여 복잡한 처리 흐름을 만듭니다.

## 핵심 속성

### 태스크 유형 (Task Type)
태스크가 어떤 종류의 작업인지 식별하는 문자열입니다. 스케줄러는 이 값을 레이어의 **입력 타입**과 매칭하여 어떤 레이어가 이 태스크를 처리할지 결정합니다. 예: `"raw"`, `"analysis_result"`

### 페이로드 (Payload)
태스크에 실려가는 실제 데이터입니다. 파일 경로 목록, 분석 결과, 사용자 입력 등 어떤 형태든 담을 수 있습니다.

### 우선순위 (Priority)
태스크의 처리 순서를 결정하는 등급입니다. 큐에서 높은 우선순위의 태스크가 먼저 꺼내집니다. 같은 우선순위 내에서는 선입선출(FIFO) 순서를 따릅니다.

- **높음(High)**: 즉시 처리 필요. 스레드 실행 시 자동 부여
- **보통(Medium)**: 기본값
- **낮음(Low)**: 여유 있을 때 처리

## 상태와 생명주기

```
                 ┌──────────┐
                 │ Pending  │ ← 큐에 진입
                 └────┬─────┘
                      │ 스케줄러가 꺼냄
                      ▼
                 ┌──────────┐
                 │ Running  │ ← 워커가 처리 중
                 └────┬─────┘
                      │
              ┌───────┴───────┐
              ▼               ▼
        ┌──────────┐    ┌──────────┐
        │   Done   │    │  Failed  │
        └──────────┘    └──────────┘
```

- **Pending**: 큐에서 실행을 기다리는 상태
- **Running**: 워커가 실제로 처리 중인 상태. 매칭된 레이어에 워커가 없으면 즉시 Failed로 전환
- **Done**: 처리가 성공적으로 완료됨
- **Failed**: 처리 중 에러가 발생함, 또는 레이어에 워커가 없어 실행 불가능함

## 파생과 깊이

태스크는 워커의 실행 결과로 **후속 태스크**를 파생시킬 수 있습니다. 이때 부모-자식 관계가 형성되며, 파생 깊이(Execution Depth)가 1 증가합니다.

```
원본 태스크 (depth=0)
  └→ 파생 태스크 (depth=1)
       └→ 파생 태스크 (depth=2)
            └→ ...
```

**최대 실행 깊이: 10** — 이 한도를 초과하면 스케줄러가 해당 태스크를 무시합니다. 이는 AI 워커가 무한히 새 태스크를 생성하는 것을 방지하는 안전장치입니다.

## 재시도

태스크는 실행 실패 시 재시도할 수 있습니다.

- **기본 최대 재시도 횟수**: 3회
- 재시도 시 새로운 태스크가 생성되어 큐에 다시 들어갑니다

## 다른 개념과의 관계

- **Layer**: 태스크의 `taskType`과 레이어의 `inputTypes`가 매칭되면 해당 레이어로 라우팅됨
- **Worker**: 레이어에 소속된 워커가 태스크를 실제로 수행함. 결과로 새 태스크를 파생시킬 수 있음
- **Thread**: 스레드 실행 시 내부적으로 태스크를 생성하여 레이어를 순차적으로 통과시킴
- **Queue**: 생성된 태스크가 대기하는 공간
- **MessageBus**: 태스크가 생성되거나 상태가 변할 때 이벤트를 발행함. 구독자 예외는 격리되며 `onError` 콜백으로 수집 가능
